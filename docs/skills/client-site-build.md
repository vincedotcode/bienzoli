# Skill: Client Site Build

## Purpose
Repeatable workflow to deliver client websites consistently from intake to archive.

## Process
1. RECEIVE — Client fills onboarding form or provides brief via WhatsApp.
2. BRIEF — Create `docs/clients/[name]/brief.md` from intake details.
3. CONTRACT — Generate contract from template, send for signature.
4. INVOICE — Generate invoice and send deposit request (50% where applicable).
5. SETUP — Create `clients/[name]/`, initialize project, configure domain plan.
6. DESIGN — Build according to package tier and design-system tokens.
7. CONTENT — Integrate client copy, photos, logos, and contact details.
8. REVIEW — Send preview link and collect consolidated feedback.
9. REVISE — Implement revisions within package limits.
10. DEPLOY — Deploy on Vercel, configure custom domain, confirm SSL.
11. VERIFY — Run PageSpeed tests (90+ target), responsive QA, link/form QA.
12. DELIVER — Send production link and request remaining payment.
13. FOLLOW-UP — Day 7 satisfaction check; day 30 testimonial + referral ask.
14. ARCHIVE — Publish case study, generate social cards, sync assets to Drive.

## Exit Criteria
- Scope delivered as contracted.
- Payment complete.
- Status log and client log updated.
