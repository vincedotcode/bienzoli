# Clients Directory Guidelines

This folder contains codebases for active and completed client projects.

## Rules
1. One folder per client: `clients/[client-name]/`.
2. Keep production code here; project documentation lives in `docs/clients/[client-name]/`.
3. Every client project must map to one package tier and one status file.
4. Include deployment notes and domain configuration in the client project README.
5. Do not store secrets in repository files.
